Project Overview

The Medication Tracking System is a Java-based console application that simulates the management of patients, doctors, medications, and prescriptions
within a pharmacy or medical clinic. The project demonstrates Object-Oriented Programming principles such as inheritance, encapsulation,
composition, and modularity.

The system allows users to:

    - Add and manage patients, doctors, and medications
    - Create prescriptions linking doctors, patients, and medications
    - Check for expired medications
    - Restock medications
    - Generate summary reports and perform searches

----------------------------------------------------------
System Architecture

| Class                  | Description          | Key Responsibilities                                           |
| ---------------------- | -------------------- | -------------------------------------------------------------- |
| `Person`               | Abstract base class  | Provides shared fields and methods for `Doctor` and `Patient`. |
| `Patient`              | Subclass of `Person` | Stores patient details, medications, and prescriptions.        |
| `Doctor`               | Subclass of `Person` | Stores doctor information and their list of patients.          |
| `Medication`           | Independent entity   | Holds medication data, including quantity and expiry date.     |
| `Prescription`         | Composition class    | Links a doctor, patient, and medication together.              |
| `MedicationTracking`   | Core manager class   | Manages CRUD operations, search, and reporting.                |
| `Menu`                 | Entry point          | Provides a console-based user interface for interaction.       |

----------------------------------------------------------
User Interface Menu

When running the application, the console presents the following options:

1.  Add A New Patient
2.  Add A New Doctor
3.  Add A New Medication To The Pharmacy
4.  Print System Report
5.  Check If Meds Are Expired
6.  Process A New Prescription
7.  Print All Scripts For Specific Doctor
8.  Restock the Drugs in the Pharmacy
9.  Print All Scripts For Specific Patient
10. Exit

Usage Instructions

1. Open a terminal or command prompt.
2. Navigate to the project folder:
       cd JAVA-GROUP4-LEAH-STEPHEN
3. Compile the program:
       javac MedicationTracking/*.java
4. Run the menu system:
       java MedicationTracking.Menu
5. Follow the on-screen instructions to interact with the system.

----------------------------------------------------------
Error Handling

- Invalid menu inputs are caught and prompt re-entry.
- Quantity and date fields are validated with appropriate messages.
- Restock and prescription functions include null and range checks.
- Expiry date detection handled automatically using `LocalDate.now()`.

----------------------------------------------------------
Conclusion

This project demonstrates a complete implementation of Object-Oriented Programming principles in Java.
It provides a realistic simulation of a pharmacy management system and is designed for maintainability and future database integration.
With a clean architecture, modular design, and intuitive user interface, this program serves as a robust foundation for future development.
